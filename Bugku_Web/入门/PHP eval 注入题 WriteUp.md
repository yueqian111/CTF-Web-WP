# PHP eval 注入题 WriteUp

## 一、题目代码

```PHP

<?php
include "flag.php";
$a = @$_REQUEST['hello'];
eval("var_dump($a);");
show_source(__FILE__);
?>
```

### 代码解析

- `include "flag.php"`：引入包含`$flag`变量的文件。

- `$a = @$_REQUEST['hello']`：接收`GET/POST/COOKIE`中的`hello`参数，`@`用于抑制错误。

- `eval("var_dump($a);")`：核心漏洞点，将字符串`var_dump($a)`作为PHP代码执行，`$a`完全可控。

- `show_source(__FILE__)`：显示当前文件源码。

## 三、关键疑问与解答

### 疑问1：为什么`?hello=system('cat flag.php')`不直接显示源码？

- **原因**：`system('cat flag.php')`输出的内容以`<?php`开头，浏览器触发容错机制，将其识别为HTML注释，在可视化区域隐藏，仅保留在页面源码中。

- **解决**：查看浏览器页面源码，或使用编码/反转义方式让内容以纯文本显示。

### 疑问2：为什么页面只显示`string(2) "?>"`？

- **原因**：`system()`函数的返回值是命令执行后**最后一行输出的字符串**。`flag.php`最后一行是`?>`，因此`var_dump()`打印的是这个返回值。

- **补充**：`system()`本身会将命令输出直接发送到浏览器，返回值仅为最后一行。

### 疑问3：为什么`system('ls')`能打印多行内容？

- **原因**：PHP的`system()`在服务器模式下，会在每行输出后自动刷新Web缓存，将输出直接打印到浏览器，而非仅返回最后一行。

- **对比**：若需获取命令原始输出，可使用`passthru()`函数。

### 疑问4：为什么PHP代码在浏览器中像注释？

- **原因**：浏览器解析HTML时，遇到`<?`开头的字符串，会触发容错机制，将其及后续内容视为注释处理，不进行渲染。

- **规范**：根据HTML解析规则，当解析器在预期标签名位置遇到`?`（U+003F）时，会将其及后续内容视为注释。

### 疑问5：为什么`file()`/Base64/`tac flag.php`能看到源码？

- **原因**：这些方式可将PHP代码转换为非PHP格式的纯文本（如Base64编码、反转义字符），避免浏览器将其识别为注释，从而直接显示内容。
