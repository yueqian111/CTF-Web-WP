## 二、关键绕过技巧详解

### 1. MD5弱类型比较绕过（两种核心方式）

#### 方式1：0e开头的MD5哈希值绕过（经典弱类型）

- **原理**：PHP中使用`==`进行比较时，会自动对字符串做**类型转换**。如果字符串以`0e`开头且后续全为数字，会被解析为「科学计数法的0」（ $0 \times 10^n = 0$ ）。因此只要两个不同字符串的MD5值都以`0e`开头，`md5(v1) == md5(v2)`就会成立。

- **经典可用值**：

    - `v1 = QNKCDZO` → MD5：`0e830737632659077758844106201202`

    - `v2 = 240610708` → MD5：`0e4628901725699156199969836901237`

- **验证**：在PHP中执行`var_dump(md5("QNKCDZO") == md5("240610708"));`，结果为`bool(true)`。

#### 方式2：MD5数组绕过（参数类型异常）

- **原理**：`md5()`函数的设计初衷是处理**字符串**，如果传入**数组**，函数会直接返回`NULL`，且不会报错。由于`NULL == NULL`，因此只要v1和v2都传数组，就能满足`md5(v1) == md5(v2)`，同时`v1 != v2`（两个不同数组）也成立。

- **参数格式**：GET请求中传递数组的格式是`参数名[]=值`（而非直接写`参数名=[值]`），例如`v1[]=1&v2[]=2`。

- **验证**：在PHP中执行`var_dump(md5([]) == md5([]));`，结果为`bool(true)`；执行`var_dump([] != []);`，结果也为`bool(true)`。

### 2. strcmp函数数组绕过（核心细节补充）

#### 原理

`strcmp($a, $b)`函数用于**逐字节比较两个字符串**，返回规则：

-  $a < $ b → 返回负数

-  $a = $ b → 返回0

-  $a > $ b → 返回正数

但如果传入的参数不是字符串（比如数组），函数会直接返回`NULL`。而PHP中`!NULL`等价于`true`，因此`!strcmp($v3, $flag)`会成立，从而绕过该判断。

#### 关键细节：为什么是`v3[]=1`而非`v3=[1]`？

- `v3=[1]`：这种写法本质是传递**字符串"[1]"** ，`strcmp`会把它当作普通字符串和`$flag`比较，除非`$flag`恰好是"[1]"，否则无法绕过；

- `v3[]=1`：这是**HTTP协议中向PHP传递数组的标准格式**，PHP会自动将`$_GET['v3']`解析为数组`array(1)`，触发`strcmp`的参数类型异常，返回`NULL`。

简单来说：`[]`是PHP识别GET数组参数的「语法标识」，直接写`[1]`只是普通字符串，无法被解析为数组。
